# Attention Is All You Need

👉阅读原文 Transformer 2017

【Motivation】：

传统的序列转换任务，如机器翻译，通常依赖于循环神经网络（RNN）或卷积神经网络（CNN）。然而，这类模型在处理序列数据时，往往需要按序计算，导致并行计算能力受限，且在捕捉长距离的依赖关系上效果不佳。因此，论文提出完全基于注意力机制的架构来解决上述问题。

【Method】：

论文提出了一种全新的，简单的神经网络架构 ——Transformer。该架构基于编码器 - 解码器结构，但完全摒弃了 RNN 和 CNN，转而采用注意力机制来捕捉输入和输出之间的依赖关系。

输入编码：Transformer 通过词嵌入技术将输入的词元转换为向量表示。为了弥补不使用循环或卷积结构可能带来的位置信息、时序特征的缺失，Transformer 还引入了位置编码（Positional Encoding），以捕捉序列中单词的顺序信息。

编码器（Encoder）：编码器由多个相同的层堆叠而成（论文中为 6 层）。每个层包含两个主要子层：多头自注意力层（Multi-Head Self-Attention）和前馈神经网络层（Feed-Forward Neural Network，FFN，本质上是 MLP）。这两个子层之间引入了残差连接（Residual Connection）和层归一化（Layer Normalization），以增强模型的训练稳定性和性能。

解码器（Decoder）：解码器的结构与编码器相似，也由多个相同的层堆叠而成（论文中为 6 层），同样有残差连接和层归一化。每层包含三个子层：带掩码的多头自注意力层、编码器 - 解码器注意力层和前馈神经网络层。使用掩码是因为需要确保在生成输出序列时，模型只能关注到当前位置之前的输入信息，从而保持训练和推理的一致性。

注意力机制：Transformer 的核心在于其丰富的注意力机制：

  • 缩放点积注意力（Scaled Dot-Product Attention）：这是最基本的注意力机制，但是考虑到键值的特征维度可能很高，为了避免 softmax 梯度消失，论文对点积计算后的结果进行了缩放，具体公式如下，其中 dkd_kdk​ 是键值的特征维度：

    Attention(Q,K,V)=softmax⁡(QKTdk)VAttention(Q,K,V) = \operatorname{softmax}(\frac{QK^T}{\sqrt{d_k}})V Attention(Q,K,V)=softmax(dk​​QKT​)V

  • 多头注意力（Multi-Head Attention）：通过全连接层重新表征查询，键和值的特征，使得模型并行地在不同的表示子空间中捕捉信息，增强了模型的表示能力。

  • 自注意力（Self-Attention）：在编码器和解码器的自注意力子层中，查询、键和值都来自同一数据序列。自注意力机制通过计算序列中每个位置的注意力权重，捕捉序列内部的依赖关系。

  • 编码器 - 解码器注意力（Encoder-Decoder Attention）：在解码器的编码器 - 解码器注意力子层中，查询来自解码器的上一层输出，而键和值则来自编码器的输出。这种注意力机制有助于生成与整个输入序列相关的输出序列。

# Improving Language Understanding by Generative Pre-Training

👉阅读原文 GPT-1 2018

【Motivation】：

在 GPT（Generative Pre-trained Transformer）模型提出之前，深度学习领域，特别是自然语言处理（NLP）方向，广泛依赖于监督学习方法，这要求大量的带标签数据来训练特定任务的模型。而大规模的带标签数据往往难以获取，标注数据需要高昂的人工成本。与此同时，一些无监督学习的案例，如预训练词嵌入模型展示了在多种 NLP 任务中良好的泛化性能。这启发研究者探索一种更通用的解决方案。

【Method】：

Transformer 架构：论文基于 Transformer 的解码器进行修改，由 12 层堆叠的 Decoder blocks 构成，每层都包含带掩码的多头自注意力和前馈神经网络。这种设计使得模型能够自回归地生成序列数据。对于不同的下游任务，解码器之后连接了不同权重的线性层，以转换模型的输出，使其符合特定任务的要求。

训练方式：论文提出了一种两阶段的训练策略：

  • 生成式预训练：第一阶段，使用大量未标注的文本数据进行无监督训练，让模型（Transformer 解码器）学习到语言的通用生成和表示能力。
  • 特定任务微调：针对特定的 NLP 任务（如文本分类、问答系统等）设计不同的模型架构，在解码器后添加线性层。这一阶段会使用少量带标签的数据进行有监督的微调，收敛速度非常快。

论文通过一系列实验验证了这种预训练 + 微调策略的有效性，也证明了 Transformer 架构零样本（zero-shot）的行为特点。这一成果不仅推动了 NLP 领域从传统的任务特定模型训练向基于大规模预训练模型的方法转变，也为后续的自然语言理解和生成技术的发展奠定了重要基础。


# BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding

👉阅读原文 BERT 2018

【Method】：

BERT 的工作机制与 GPT 有相似之处，其核心策略是在大规模未标记的文本数据上进行预训练，以学习语言的通用特征。随后，针对特定的下游任务，设计单独的模型结构，并使用预训练得到的参数进行初始化。接着，根据有标记的数据对这些参数进行微调，以适应特定任务的需求。

不同之处在于，BERT 采用的是多层双向 Transformer 编码器架构，通过双向的注意力机制同时关注输入文本左右两侧的上下文信息，从而更准确地理解每个单词的语义。

词向量表示：BERT 的输入表示不仅包含了传统的词向量，还引入了两种额外的向量。位置词向量：用于表示单词在句子中的位置信息，类似于 Transformer，但 BERT 使用的是可学习的位置向量。句子词向量：用于区分不同的句子，有助于模型理解不同句子之间的界限和关系。这三种向量相加后，构成了最终输入 BERT 模型的词向量表示。

预训练：BERT 在预训练阶段采用了两个有监督任务：

  • Masked Language Model（MLM）：一种掩码语言模型的预训练方法，即用特殊标记 [MASK] 随机掩盖输入句子中的一些单词，然后让模型预测这些掩盖单词的原始词汇 ID，迫使模型学习到每个单词的上下文表示。
  • Next Sentence Prediction（NSP）：BERT 在预训练时，需要判断两个给定的句子是否是连续的文本，有助于模型学习句子间的逻辑关系，对于理解复杂文本结构至关重要。

微调：训练具体任务时，需要根据任务的特性设计相应的输入和输出格式。然后，使用预训练得到的 BERT 模型作为起点，继续调整模型的参数，使得 BERT 能够灵活地适应各种自然语言处理任务。

# Language Models are Unsupervised Multitask Learners

👉阅读原文 GPT-2 2019

【Motivation】：

传统机器学习系统在处理自然语言处理任务时，通常采用有监督学习的方法，即便在预训练 - 微调架构下，也需要对特定任务进行架构设计和有监督的微调。作者认为，这种方式使得系统对数据分布和任务规范非常敏感，导致在面对数据或任务规范的微小变化时，性能会显著下降，缺乏泛化能力。因此，作者希望构建自然语言处理领域更为通用的系统，使其能够在无需任何任务特定监督的零样本条件下，执行多项下游任务，提高系统的泛化性和适应性。

【Method】：

为了摒弃预训练模型需要针对特定任务的微调步骤，论文修改了模型的输入，增加任务维度的描述，从 P(output∣input)P(output\mid input)P(output∣input) 改为 P(output∣input,task)P(output\mid input, task)P(output∣input,task)。为了避免每增加一个任务就得添加一部分训练集，GPT-2 简单粗暴地扩大了语料库，尽可能覆盖所有任务。具体来说，作者从互联网上抓取了大量高质量的文本数据，构建了名为 WebText 的大型数据集。GPT-2 不需要刻意地组织训练集的格式，只需要将海量的数据直接喂给模型，模型便能在学习过程中自然捕捉到各种任务的规律和特征。在预测时，只需要通过一些提示（Prompt），便能得到相对准确的输出。

输入表征：GPT-2 在输入表征方面结合了词级模型和字节级方法的优点，采用了字节对编码（BPE）作为折中方案。为了避免直接应用字节序列可能导致的词汇合并不理想的问题，GPT-2 对 BPE 进行了限制，禁止其在字节序列中跨字符类别进行合并。这一改进使得 GPT-2 能够有效地对任何 Unicode 字符串进行概率计算。

模型架构：在 GPT-1 的基础上，GPT-2 进行了一些调整。首先，将后置层归一化改为前置层归一化，即在数据处理之前进行归一化操作。其次，在模型最后一个自注意力块后额外增加了一个层归一化层。此外，GPT-2 改进了模型参数的初始化方法，即将残差层的权重缩放因子设置为1N\frac{1}{\sqrt{N}}N​1​，其中 NNN 代表残差层的数量。同时，GPT-2 还扩大了词典大小（从 GPT-1 的 40000 扩大到 50257），增加了上下文长度（从 512 变为 1024），并调整了批量大小（ batchsize 变为 512），以适应更大规模的数据集和更复杂的任务需求。

GPT-2 的工作证明了语言模型具有零样本迁移学习的能力，展现了大规模语言模型的巨大潜力和广泛的应用前景。

# Megatron-LM: Training Multi-Billion Parameter Language Models Using Model Parallelism

👉阅读原文 Megatron-LM 2019

【Motivation】：

随着在大型语言模型上使用大规模数据进行预训练的暴力方法逐渐出圈后，一个很现实的问题是如何满足这样规模庞大的训练任务对显存和计算资源的需求（这可就是 NVIDIA 的老本行了）。本论文提出使用层内模型并行的方式来提高训练大规模语言模型的效率，从而有效地应对显存和计算资源限制的问题。

【Method】：

论文针对基于 Transformer 架构的网络进行实验，提出的并行方案也主要针对于 Transformer 架构。

前馈神经网络：Transformer 中，前馈神经网络就是一个 MLP。论文使用的非线性激活函数是 GeLU，最后接上 Dropout 层。前馈神经网络的表示（不考虑偏置项）如下：

FNN⁡(X)=Dropout⁡(GeLU⁡(X⋅A)⋅B)\operatorname{FNN}(X) = \operatorname{Dropout}(\operatorname{GeLU}(X \cdot A) \cdot B) FNN(X)=Dropout(GeLU(X⋅A)⋅B)

其中 A∈Rh×4h,B∈R4h×hA \in \mathbb R^{h \times 4h}, B \in \mathbb R^{4h\times h}A∈Rh×4h,B∈R4h×h 分别是两个线性层的权重参数。为了实现并行计算，论文对线性层的参数进行了分割。具体来说，对参数矩阵 AAA 和 BBB 分别按列和按行进行切分，然后数据分别在多个 GPU 上作并行计算。这种分割方式可以避免在第一个全连接层之后合并数据，从而避免 GPU 间的通信（All-Reduce）。

自注意层：每一个注意力头本身具有很好的并行性。对于每个头的 Q、K、VQ、K、VQ、K、V 参数，以及后面的全连接层权重 BBB，可以依据上述分割方式分成多份，分布到不同的 GPU 上进行计算。

词嵌入：语言模型的输入和输出需要使用一个相同的词嵌入矩阵 V×HV \times HV×H。当词表（VVV）非常大时，这个矩阵会占用大量的显存资源。因此，论文对词嵌入矩阵按行进行切分，每一片只包含部分词表信息，并分别存放在不同的 GPU 上。当然，这可能导致某些情况下无法找到对应的词嵌入表示，此时不得不采用 All-Reduce 通信操作来同步不同 GPU 上的词嵌入信息。

此外，论文还对交叉熵损失进行了优化，并让每个 GPU 自己执行残差连接和层归一化操作，以减少通信开销，提高计算效率。

总而言之，论文通过数据和模型的并行设计，尽可能地减少 GPU 的通信开销，提高并行计算的效率，也有助于解决显存不足的问题。

# Exploring the Limits of Transfer Learning with a Unified Text-to-Text Transformer

👉阅读原文 T5 2019

【Method】：

论文中提出的模型框架被称为 Text-to-Text Transfer Transformer（T5），它采用的是 Transformer 的编码器 - 解码器架构。这种架构与以往的研究不同，以往的研究可能只使用编码器或只使用解码器。T5 选择这种编码器 - 解码器架构，能够有效执行论文提出的文本到文本任务（Text-to-Text）。

模型的训练过程分为预训练和微调两个阶段：在大规模无标签数据上做预训练，使用的训练任务类似于 MLM；然后针对文本到文本任务进行微调。文本到文本任务是本篇论文的核心方法，即在原始输入序列前增加一个与任务相关的文本前缀，从而将所有 NLP 任务都转化成文本到文本任务。这其实与 GPT-2 的思路有异曲同工之处，类似的想法在前人的工作中也看得到。

事实上，这篇论文并没有提出新颖的创新点。但它进行了大量的实验，对比了不同的预训练目标、模型架构、无标签数据集等设置下的模型在不同 NLP 任务下的性能表现。此外，论文还开源了预训练模型代码和权重，以及训练数据集，为后续研究工作提供了宝贵的经验和见解。


# Language models are few-shot learners

👉阅读原文 GPT-3 2020

【Motivation】：

GPT-2 提出了无监督语言模型的概念设想，但其有限的性能表现并没有带来突破性的成功。为了进一步探索大规模语言模型的潜力，GPT-3 继续沿着这一方向深入，在 "one-shot"、"few-shot" 和 "zero-shot" 学习条件下对语言模型的性能表现进行实验评估。

【Method】：

GPT-3 使用的模型架构与 GPT-2 类似。作者一共训练了八个规模的语言模型，参数从 1.25 亿到 1750 亿不等。针对这些不同规模的语言模型，作者进行了多项实验，分别比较它们在 "one-shot"、"few-shot" 和 "zero-shot" 的设置下的表现。

few-shot：模型在推理时会接收到一些示例数据作为上下文，这些示例展示了如何根据输入生成期望的输出。

one-shot：与 few-shot 类似，但只提供一个示例数据。

zero-shot：模型仅接收到任务输入，而不提供任何示例数据作为上下文。

实验结果表明，随着模型参数量的增加，模型在少样本学习条件下的性能得到显著提高。与 "one-shot" 和 "zero-shot" 设置相比，"few-shot" 的性能通常更高，且随着模型容量的增大，这种性能差距也变得更加明显。

从现代大模型运作模式的角度来看，GPT-3 的工作为通用语言模型和 prompt 设计提供了重要的研究基础。它展示了大规模语言模型在少样本学习条件下的强大潜力，以及通过适当的 prompt 设计来引导模型生成期望输出的可能性。


# Switch Transformers: Scaling to Trillion Parameter Models with Simple and Efficient Sparsity

👉阅读原文 Switch Transformers 2021

【Motivation】：

MoE（Mixture-of-Experts）模型通过引入稀疏性设计，使得不同输入数据能够通过选择不同 “专家” 进行计算，从而在有限的计算资源下实际出更大规模的模型。而在大语言模型竞争激烈的背景下，Google 研究人员将 MoE 理念应用于 Transformer 架构，催生了本论文的研究成果。

# Evaluating Large Language Models Trained on Code

👉阅读原文 Codex 2021

【Motivation】：

近年来，大语言模型在各类 NLP 任务上取得了显著成就。鉴于这些模型在理解和生成文本方面的强大能力，OpenAI 瞄准了语言模型在代码程序相关任务中的潜在应用，进行了深入研究，并据此推动了早期产品 Github Copilot 的开发。

【Method】：

模型架构：这篇论文主要关注根据文档提示（docstrings）生成 Python 函数的任务。为了实现这一目标，论文提出了 Codex 模型。Codex 模型使用预训练的 GPT-3 模型的参数进行初始化，并在 GitHub 上开源的大量代码数据上进行微调，同时在人工手写的 HumanEval 数据集上进行验证评估。

评估方法：为了评估模型生成代码的能力，论文基于功能正确性的原则，设计了一种特定的评估方法，使用以下公式来计算模型在特定问题上的通过率：

pass⁡@k:=E Problems [1−(n−ck)(nk)]\operatorname{pass} @ k:=\underset{\text { Problems }}{\mathbb{E}}\left[1-\frac{\binom{n-c}{k}}{\binom{n}{k}}\right] pass@k:= Problems E​[1−(kn​)(kn−c​)​]

其中 kkk 是设定的模型为每个任务生成的回答个数，nnn 是模型实际生成的回答总数（通常 n>kn\gt kn>k），而 ccc 是这 nnn 个回答中通过测试的正确答案数量。这个公式提供了一个无偏估计，用于评估模型生成可通过测试的正确回答的概率。

纵观整篇文章，虽然它并没有令人眼前一亮的创新点，但其对大语言模型在编程方面的相关问题进行了深入研究和探索，也为未来语言模型在更广泛领域的应用提供了有益的参考。

# Finetuned Language Models are Zero-Shot Learners

👉阅读原文 FLAN 2021

【Motivation】：

GPT-3 的工作展示了语言模型在 "few-shot" 设置下的卓越表现，然而该模型在 "zero-shot" 设置下表现相对逊色。

作者认为，这种性能差异的主要原因在于模型在推理时接收到的提示（prompt）与其预训练期间接触的数据格式存在显著不同。为此，论文作者提出了一种名为指令微调（instruction tuning）的方法，旨在提升语言模型的 "zero-shot" 性能。

【Method】：

作者在一个拥有 137B 参数的预训练模型 LaMDA-PT 的基础上，使用一个包含超过 60 个 NLP 数据集的混合数据集进行了微调。这些数据集都是通过自然语言指令来表达的，这一过程被称为指令微调。经过这一过程的模型被命名为 FLAN（Finetuned Language Net）。

在指令微调过程中，作者采用了如图示例所示的模板来构造训练数据。这些模板包含了各种自然语言指令，帮助模型学会理解和响应各种任务要求。

此外，FLAN 的调优过程在不同的任务集群上进行，接着在未被微调的任务上进行测试，从而检验指令微调是否增强了模型对未见任务的泛化能力。

相比于 GPT-3 直接在预训练模型上输入任务提示，经过指令微调的模型能够更好地理解和响应自然语言指令，从而更好地应用于下游任务，还能提升在未见过的任务上的性能表现。


# Multitask Prompted Training Enables Zero-Shot Task Generalization

👉阅读原文 T0 2021

【Motivation】：

相关研究表明，大语言模型在多样化的自然语言处理任务上展现出了令人瞩目的零样本泛化能力。这种能力通常归因于模型在预训练阶段隐式地学习到了多任务的知识。然而，研究者好奇，是否可以通过显式地引入多任务学习机制，进一步提升模型的零样本泛化能力。为了探究这一问题，作者提出 T0 模型，研究了多任务提示训练策略对于增强语言模型零样本泛化能力的潜在影响。

【Method】：

论文在一个预训练好的、基于编码器 - 解码器架构的 T5+LM 模型（即在预训练的 T5 模型上进一步进行了 LM 训练的变体）上，使用一个大型的、包含多种任务类型的、带有自然语言提示词的数据集进行微调。这个多任务数据集，来自作者收集的共计 177 个不同任务的数据集，使用人工设计的提示词模板统一转换成相应的 prompt 格式。

在微调阶段，模型在数据集的子集上进行训练，然后测试其在未见过的任务上的表现。这一思想很大程度上也与 FLAN 相同。

这篇论文与先前的工作有太多相似的地方，但它对模型微调提升零样本表现进行了更深入的研究和探索，具有一定的参考价值。

# GLaM: Efficient Scaling of Language Models with Mixture-of-Experts

👉阅读原文 GLaM 2021

【Motivation】：

Google 研究人员提出了名为 GLaM（Generalist Language Model）的模型，通过引入 Mixture-of-Experts（MoE）稀疏激活网络架构，成功扩展了语言模型的参数规模，同时显著降低了所需的计算资源。可以发现，GLaM 在这一方面的设计与 Switch Transformer 有着相同之处，都依靠稀疏激活来优化计算效率。

【Method】：

模型架构：GLaM 模型基于 Transformer Decoder 结构，并在此基础上进行了以下修改：

  • MoE 子块：对于前馈神经网络层，GLaM 改用包含 64 个前馈网络专家的 MoE 层。这些专家的选择依据一个门网络和随后的 softmax 函数来计算分布概率。在做前向推理时，每个 token 仅激活其中两个专家。这种稀疏激活机制有效减少了模型的计算量。
  • non-MoE 子块：在 GLaM 的非 MoE 子块中，模型使用了 GLU（Gated Linear Unit）来替换第一个线性层，并采用了 GeLU（Gaussian Error Linear Unit）作为激活函数。
  • 位置嵌入：GLaM 模型使用了每层的相对位置偏置来替换传统的标准位置嵌入。这一改动有助于模型更好地捕捉序列中的位置关系。

实验：论文对预训练后的 GLaM 模型进行相关的评测，主要关注其在零样本、单一样本以及少样本情况下的表现，并与 GPT-3 进行了对比。实验结果表明，GLaM 在保持高性能的同时，显著降低了计算成本。

# WebGPT: Browser-assisted question-answering with human feedback

👉阅读原文 WebGPT 2021

【Motivation】：

长形式问题回答（Long-Form Question-Answering, LFQA）是 NLP 领域一项新兴挑战，它要求模型能够针对开放式问题生成长段且详细的答案。LFQA 与传统问答任务的主要区别在于其答案的长度和复杂性，通常涉及信息检索和信息合成两个核心步骤。由于当前技术在此类任务上的表现远未达到人工水平，论文希望通过结合模仿学习和强化学习，在基于文本的网络浏览环境中对模型进行训练和微调，从而提升长形式问题回答的质量。

【Method】：

基于文本的网络浏览环境：不同于文档检索的方法，论文采用 Microsoft Bing 搜索引擎作为信息检索工具。模型通过执行如下指令与搜索引擎交互，提取相关上下文的摘要并记录来源信息。这些信息与原始任务输入合并，形成一个完整的 prompt 输入到模型中。

数据集：作者设计了一个图形化用户界面，收集人类使用浏览器回答问题的行为数据，这些数据被称为演示数据（demonstrations）。此外，作者还收集了人类评估者对模型生成的多个答案的评估结果，这些数据被称为比较数据（comparisons）。演示数据展示了人类如何使用搜索引擎来查找和整合信息，比较数据用于评估不同答案的质量，并为奖励模型的训练提供标签。

训练细节：论文使用预训练的 GPT-3 模型，利用其零样本行为的能力来理解搜索引擎提供的上下文信息。模型的训练过程包括以下四个方面：

  1. 行为克隆（Behavioral Cloning, BC）：利用收集到的演示数据作为标签微调模型，使其能够模仿人类使用浏览器检索和整合信息的行为。
  2. 奖励建模（Reward Modeling, RM）：基于 BC 模型训练一个奖励模型。该模型接受问题和带有引用的回答作为输入，输出一个标量奖励值，用于评估回答的质量和人类偏好程度。奖励模型的训练使用比较数据作为标签，并采用交叉熵损失函数进行优化。
  3. 强化学习（Reinforcement Learning, RL）：使用 PPO 强化学习算法对 BC 模型进行微调。在强化学习环境中，模型根据奖励模型给出的分数（加上 KL 散度惩罚以避免过度优化）进行策略更新。
  4. 拒绝采样（Rejection Sampling, best-of-n）：从 BC 模型或者 RL 模型中采样固定数量的答案，然后选择一个奖励模型评分排名最高的答案。

# Improving language models by retrieving from trillions of tokens

👉阅读原文 Retro 2021

【Motivation】：

在 NLP 领域，当前的大型语言模型使用海量参数和大规模数据集进行训练，取得了不错的效果。然而，这些模型的性能增强往往伴随着计算资源的巨大消耗。这篇论文希望探索一种新的方法，即在不大幅增加模型参数数量和计算负担的前提下，通过引入大规模的外部记忆来增强语言模型的能力。

【Method】：

论文提出了检索增强的自回归语言模型 ——Retrieval Transformer（Retro）。Retro 模型能够在包含万亿级 token 的大规模数据库中进行检索，并结合检索到的信息与输入的序列数据进行推理。具体来说，它是基于前面的 token 和 从数据库中检索到的相关文本分片来计算下一个 token 的似然概率。

检索数据库：Retro 使用的检索数据库的是一个键值型数据库。其中，键是冻结的 BERT 模型的嵌入向量，而值是从训练数据集中采样的连续文本分片，并以第一个分片的嵌入向量作为其键。为了避免在检索过程中违反因果关系，即模型在训练时不应当访问到未来分片的信息，模型会过滤掉那些与训练序列来自同一文档的邻居分片。

近邻检索：在模型的前向推理阶段，输入序列被划分为等长的分片。对于每个分片，Retro 都会在数据库中检索与该分片最相似的 k 个邻居分片。为了实现高效的最近邻搜索，论文采用了 SCaNN 库，通过计算输入序列分片嵌入向量与数据库中键的 L2L_2L2​ 范数距离来找到最相似的邻居。

模型架构：Retro 基于 Transformer 编码器 - 解码器架构。编码器负责处理检索到的邻居分片，并将它们转换成编码表示。解码器则利用这些编码表示来生成序列的下一个分片。在 Retro 中，解码器由 Retro 块和标准 Transformer 块交替组成。

在解码器中，Retro 模型引入了分片交叉注意力（Chunked Cross-Attention, CCA）机制，使得模型在生成每个分片时，同时考虑前一个分片的上下文信息和检索到的邻居信息。

# Chain-of-Thought Prompting Elicits Reasoning in Large Language Models

👉阅读原文 COT 2022

【Motivation】：

扩大模型规模和增加训练数据集是提升语言模型能力一种直接有效的方法，但在诸如算数、推理等任务中，单纯依靠规模和数据量的增加所带来的性能提升并不显著。研究者希望从两个方面着手提高语言模型的推理能力：第一，在模型生成答案的过程中，鼓励其给出理由和推断过程，以此引导模型得出更加准确和合理的答案；第二，利用少样本上下文（few-shot），让模型能够更快速地适应和学习新的下游任务。基于这两种思路，论文提出了一种名为 chain-of-thought prompting（思维链提示词）的方法，提高语言模型的推理能力。

【Method】：

思维链提示词，即在提示词中给出一些任务相关的基本样例，展示这些样例的中间推理步骤，从而引导语言模型生成最终回答前给出一系列具体的推理步骤。这种方法有诸多优点：

  1. 通过将复杂的推理计算过程拆分成多个相对简单的子问题，每个子问题都更容易被语言模型理解和处理，这有助于调动语言模型的推理能力。
  2. 由于思维链提示词要求模型展示推理步骤，因此当答案出现错误时，可以更容易地定位到错误的推理环节。这有助于后续对模型的改进和优化。
  3. 使用提示词的方法避免了在预训练模型阶段对训练数据和训练任务做过多的调整。只要通过适当的提示词设计，就可以利用思维链提示词来提高模型的推理能力。

论文提出的方法虽然看似简单，但其背后的思想却极具启发意义。通过在多个推理任务上的实验验证，研究者们证明了这种策略的有效性。
